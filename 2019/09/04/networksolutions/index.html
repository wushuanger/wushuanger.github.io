keep domains on networksolutions, not continue their hosting and mailbox service

wushuang
2019-09-04
hosting
SSL, custom domain, github pages, mailgun, networksolution

our hosting, domain, email service on networksolutions will be expired at the end of this month, the renew price is too much for us. explore infomations and make a reasonable choice.

Domain

keep domains on networksolutions, $11.99/year/domain(at the first time they offer us $40/year/domain😂).

Hosting

we only have one small website online at this moment, we decide not to continue their hosting service. transfer it to my VPS or even github pages. (here i transfor the manova.com and nubrikon.com to github pages)

setting DNS for website transformation

1. According to networksolution’s tuit, failed.

First, edit the A Records by selecting Edit A Records.

For the “@ none” enter the following IP address 205.178.189.129(github ip: 192.30.252.153), clear the records for the “www” and “* (All others)”, then select Continue

Second, edit the Host Aliases (CNAME records) by selecting Edit CNAME Records.

Enter “www” in the Alias column. Leave TTL set to 7200 but on the same line, enter the name servers provided by the blogging service in the Other Host section (for blogger.com, this name server would be ghs.google.com)(change to: githubusername.github.io), failed.

according to tuit below on freedom, i changed githubusername.github.io to mydomain.com. after about one hour, it works.

2. according to this tuit on freedom, it works.

on Freedom(here on Networksolutions), add an A Record and a CNAME Record.

on github pages, add “www.mydomain.com".

SSL Github Pages

according to this tuit, successfully add SSL to mydomain.com(not working for domain like .tk HA)
这里需要将a记录改成：

185.199.108.153
185.199.109.153
185.199.110.153
185.199.111.153

我说怎么manova.com的a记录是这样的呢。但是按照这个改我nubrikon.com不能访问诶。再试一下，感觉多了几条。

Email

  • netwoksolutions的邮箱服务不再继续。暂定用mailgun转发gmail。
  • 看了Using Mailgun to Setup a Professional Custom Email for Free in Gmail,觉得这个办法可行啊。
  • 但因为mailgun需要绑定信用卡，所以找了其他服务看看行不行。试了sendgrid，参考Using a Custom Domain in Gmail for Free with Mailgun and Sendgrid，在google有一部卡住了，这个文章也没提到这个，anyway，sendgrid好像必须先有具体邮箱地址才可以呀，mailgun是直接以domain为单位，指定了域名，再在mailgun里create具体邮箱？确定是这样的话，还是mailgun合适咯，到年底再explore看看。。而且这篇文章还提到mailgun的免费用户（好像是说的没有绑定信用卡的用户？）在发邮箱的时候会让对方确认是否收来自mailgun的邮件？这个有点不顺畅了，这里有个问题就是，我注册mailgun的时候就是需要绑定信用卡的，没有不绑定这个选项啊，所以这里说的信息可能是过期信息，绑定信用卡应该很有可能就不会收到这个信息。到时候再确认下（比如到文章中提到的那个人的空间）。
  • 这个视频https://www.youtube.com/watch?v=7V3m2kbXyO0 中途没有错误，但是最后没有收到邮件，用network给的那个可以收到。

最后用mailgun解决。cratolmail@gmail.com转发接收我的和其他公司邮件。cratoljohn@gmail.com给jj用。
